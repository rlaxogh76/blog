함수형 컴포넌트에서 상태를 관리하기 위한 훅.
`
useState는 아래와 같이 구성됨.

```jsx
const [state, SetState] = useState(initialValue);
```

state : 현재 컴포넌트가 기억하고 있는 **상태 변수**  
setState : 상태 변수를 새로운 값으로 업데이트하고 리렌더링을 발생시키는 **상태 변경 함수**  
initialValue : 현재 상태의 **초기값**. 숫자, 문자열, 객체, 배열 등 형태를 가질 수 있으며, 함수 전달을 통해 초기값을 가지는 것 또한 가능.

## 왜 사용할까?

함수형 컴포넌트는 렌더링될 때마다 함수가 재실행 되는데, 이를 변수만으론 상태 유지가 힘듦.  
상태 기억을 통해 상태가 변경되면 컴포넌트가 리렌더링되어 화면에 최신 데이터를 즉각적으로 보여줄 수 있음.

## 주요 특징

- **렌더링 간 데이터 유지**

  > 일반 자바스크립트 변수는 컴포넌트가 다시 렌더링되면 초기값으로 리셋됨.  
  > useState를 사용하면 컴포넌트가 여러 번 다시 렌더링되어도 이전 상태 값을 안전하게 보존 가능.

- **컴포넌트 단위의 상태 독립성**

  > 동일한 컴포넌트를 화면에 여러 개 사용하더라도,  
  > 각각의 컴포넌트는 서로 다른 useState 상태 값을 가지므로 서로 간섭하지 않음.

- **함수 컴포넌트에서 상태 관리**

  > 클래스 컴포넌트에서만 가능했던 상태 관리를 함수 컴포넌트에서도 가능하게 해준다.

- **자동 UI 업데이트**

  > 상태가 변경되면 React가 자동으로 해당 컴포넌트를 다시 렌더링하여 UI를 최신 상태로 유지해준다.

- **초기값 설정**

  > useState를 호출할 때 초기 상태 값을 설정해줘야 한다.

- **불변성 유지**

  > 상태를 직접 수정하면 React가 변화를 감지하지 못하므로,  
  > 반드시 갱신 함수를 통해 새로운 상태를 주입해야 함.

- **비동기적/일괄 처리**
- 여러 개의 상태 변경 요청이 들어와도 리액트는 즉시 반영하지 않고, 한 번에 모아서 처리함. -> 성능 최적화 가능

## 동작 원리

useState는 컴포넌트가 렌더링될 시, 리액트 내부 메모리에 상태 저장, 처음 렌더링될 때 초기 상태 값을 받아 내부적으로 관리함.

## 기본 사용법

```tsx
import { useState } from "react";

const Component1 = () => {
  const [value, setValue] = useState(0); // ()안에는 0, "", null 등이 올 수 있다.
};
```

## 사용 시 주의점

### 1. 상태는 즉시 변경되지 않는다.

React는 성능 최적화를 위해 상태를 즉시 변경하지 않고, 모아서 처리한다.  
만약 바로 변경되길 원한다면 아래처럼 함수형 업데이트를 사용하자.

```jsx
setCount(count + 1); // 잘못된 방식 : 이전 state가 즉시 반영되지 않음

setCount((prev) => prev + 1); // 올바른 방식 : 이전 상태를 함수 인자로 안전하게 참조
```

### 2. 직접 수정해서는 안된다.

상태값을 직접 변경하여 사용한다면, React는 상태 변화를 감지하지 못해 렌더링이 발생하지 않는다.  
항상 반드시 set 함수를 통해 상태를 변경하자.

```jsx
// 새로운 객체 state를 교체함.
setForm({
  ...form,
  firstName: "Taylor",
});
```

### 3. 초기값 지연 초기화

초기값 설정에 복잡한 연산이나 무거운 작업이 포함되어 있다면 매 렌더링마다 불필요하게 실행된다.  
화살표 함수를 전달하여 최초 1회만 실행되도록 최적화하자.

```jsx
const [data, setData] = useState(computeHeavyData()); // 비효율적: 매 렌더링마다 무거운 함수 실행

const [data, setData] = useState(() => computeHeavyData()); // 효율적: 최초 렌더링 시에만 실행됨
```

### 4. 최상단 레벨에서만 호출

조건문(if), 반복문(for) 내부 또는 일반 중첩 함수 내에서 useState를 호출하면 훅의 호출 순서가 보장되지 않아 에러가 발생한다.  
항상 컴포넌트의 최상단에서만 호출하자.
