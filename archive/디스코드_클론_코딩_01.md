## 레이아웃 분석하기 🎨

이번 글에선 디스코드 클론 코딩을 위한 코드를 짜기 전에 앞서, 디스코드의 레이아웃을 분석해보는 시간을 가져볼려고 합니다.

전체적인 구조를 크게 나누어보면 다음과 같이 구성되어 있었습니다.

```
서버 목록
    -> 채널 및 메뉴 영역
        -> 메인 콘텐츠 영역
            -> 추가 정보 및 활동 영역
```

좌측에서 우측으로 갈수록 더 자세한 정보가 표시되는 형태이며, 사용자는 자연스럽게 현재 보고 있는 콘텐츠에 집중할 수 있도록 설계되어 있었습니다.

### 1. 서버 목록 영역

가장 좌측에는 서버 목록이 위치해 있습니다.

디스코드 전체에서 서버 전환하는 역할을 담당하고 있습니다.

특징으로는 다음과 같은 요소들이 존재했습니다.

> 1. 원형 기반의 서버 아이콘
> 2. 매우 좁은 너비
> 3. 텍스트 없이 아이콘만 표시
> 4. 현재 선택된 서버 강조
> 5. hover 시 부드러운 애니메이션 제공

특히 서버 아이콘은 텍스트 대신 이미지 중심으로 구성되어 있었는데, 이는 사용자가 서버를 이름보다 시각적으로 기억하기 쉽기 때문이라고 느껴졌습니다.

또한 선택된 서버는 배경 강조 및 활성화 표시를 통해 현재 위치를 쉽게 파악할 수 있도록 구성되어 있었습니다.
