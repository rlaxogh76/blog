## 시작하기에 앞서

이 글에는 코드 관련 내용은 포함되어 있지 않습니다.  
개발 과정을 보고 싶은 분들께서는 [다음 글](디스코드_클론_코딩_01.md)을 방문해주세요!

## 왜 만들게 되었나? 🤔

기존 팀 프로젝트를 진행하면서 문득 이런 생각이 들었습니다.

> "너무 AI에 의존하고 있는 건 아닐까..?  
> 뭔가 성장하고 있다는 느낌이 잘 안 드는 것 같은데.."

말 그대로 AI에 지나치게 의존하다 보니,  
내가 정말 원하는 방향의 개발자가 맞는지 고민하게 되었습니다.

이런 복잡한 마음을 정리해보고자, 이번 **클론 코딩 프로젝트**를 시작하게 되었습니다.

**디스코드**를 선택한 이유는 단순했습니다.  
예전부터 클론 코딩 프로젝트를 제대로 완성해보고 싶었고,  
평소 자주 사용하는 서비스이기도 했기 때문입니다.

또한 디스코드는 **UI/UX**가 깔끔하고,  
직접 구현해보면 재미있을 것 같다는 생각도 들었습니다. 🤩

## 어떻게 만들 것인가? 🚧

물론 AI에게 단순히 "디스코드처럼 만들어줘" 라고 요청할 수 있겠죠.

하지만 이번 프로젝트의 목적은 단순 결과물이 아니라,  
제 스스로 성장하는 과정에 더 가깝습니다.

그래서 AI를 참고하더라도,  
전체적인 설계와 개발 흐름, 구현, 배포 과정까지는 최대한 직접 진행해보려고 합니다.

## 개발 흐름은 어떻게 되는가? 📝

현재 생각하고 있는 전체적인 개발 흐름은 아래와 같습니다.

> 1. 레이아웃 구성
> 2. Mock 데이터 기반 UI 구현
> 3. 상태 관리 연결
> 4. API 연결
> 5. 실시간 기능 구현
> 6. 고급 기능 구현
> 7. 보너스 기능 추가

물론 개발을 진행하면서 세부적인 방향은 바뀔 수도 있겠지만,  
큰 흐름은 아마 이런 방식으로 진행되지 않을까 싶습니다.
